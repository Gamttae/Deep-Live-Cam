# Item Finder Beacon — 안드로이드 BLE 비콘 앱

> 공기계(쓰지 않는 안드로이드 폰)를 물건에 부착해 **BLE 비콘**으로 사용하는 앱.  
> 본체(Arduino + Flipper Zero)가 이 비콘의 신호로 물건 위치를 추적합니다.

## 1. 개요

별도의 BLE 비콘을 구매하지 않고 **집에 굴러다니는 공기계**를 활용해 비콘을 만듭니다.

**장점**
- 추가 비용 없음 (안 쓰는 폰 재활용)
- 큰 배터리 → 오래 동작 (충전기 꽂아두면 무제한)
- 화면으로 어떤 물건인지 보임 ("리모컨", "지갑" 등)

**단점**
- 폰 크기 → 작은 물건엔 부적합
- 화면 켜져 있어야 안정적 (전원 연결 권장)

## 2. 작동 원리

```
[안드로이드 공기계]
   │
   │ BLE 광고 송신:
   │   - 디바이스 이름: "IF-리모컨"
   │   - 서비스 UUID:   12fe0001-cb0b-4703-bfc1-3940df000001
   │
   ▼ (무선 BLE 신호)
[Flipper Zero]
   │
   │ 광고 데이터 파싱 → "IF-"로 시작하는 이름 검출
   │ RSSI 측정 → UART로 Arduino 전송
   │
   ▼ (UART)
[Arduino Mega]
   │
   │ 서보 회전 + 비프 → 물건 방향·거리 안내
```

**핵심**: 이 앱은 **포그라운드 서비스**로 동작하므로 화면을 꺼도 광고가 계속됩니다.

## 3. 요구 사항

### 안드로이드 기기

| 항목 | 최소 | 권장 |
|---|---|---|
| Android 버전 | 6.0 (API 23) | 8.0+ |
| BLE 광고(Peripheral) 지원 | 필수 | — |
| 충전 케이블 | — | 권장 (장시간 사용) |

### BLE 광고 지원 여부 확인

앱 실행 시 자동으로 확인하지만, 미리 확인하려면:

```
설정 → 휴대전화 정보 → BLE 광고 지원 여부 확인
```

또는 Play 스토어 **"BLE Checker"** 앱 등으로 `isMultipleAdvertisementSupported` 항목 확인.

> 안드로이드 5.0(롤리팝) 이상 대부분 지원하지만,  
> 일부 저가형/오래된 단말은 미지원입니다.

### 개발 도구 (빌드 시)

- Android Studio Hedgehog (2023.1) 이상
- JDK 17
- Android SDK 34

## 4. 빌드 방법

### Android Studio로

1. Android Studio 실행 → `Open` → `item-finder/android_beacon` 폴더 선택
2. Gradle 동기화 자동 시작 (1~2분 대기)
3. 메뉴 `Build → Build Bundle(s)/APK(s) → Build APK(s)`
4. 빌드 완료 후 `app/build/outputs/apk/debug/app-debug.apk` 가 생성됨

### 명령줄로

```bash
cd item-finder/android_beacon
./gradlew assembleDebug
```

> 최초 빌드 시 `gradlew` 가 없으면 Android Studio에서 한 번 빌드한 뒤 사용.

### APK 설치

USB 디버깅 켠 후:
```bash
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

또는 APK 파일을 폰에 복사 후 직접 설치.

## 5. 사용 방법

### 5.1 첫 실행

1. 앱 실행 → 권한 요청 화면
   - **블루투스 (광고/연결)** 권한 → 허용
   - **알림 권한** (Android 13+) → 허용
   - **위치 권한** (Android 11 이하) → 허용
2. 메인 화면 표시

```
┌───────────────────────────────┐
│      Item Finder 비콘         │
│                               │
│ 이 앱을 실행한 안드로이드를   │
│ 물건에 부착하면 본체가        │
│ BLE 신호로 추적합니다.        │
│                               │
│ 물건 이름                     │
│ ┌─────────────────────────┐  │
│ │ 리모컨                  │  │
│ └─────────────────────────┘  │
│ 광고 이름은 "IF-<물건이름>" │
│ 형태로 송신됩니다.            │
│                               │
│ [광고 시작]  [광고 중지]      │
│                               │
│ 상태                          │
│ ┌─────────────────────────┐  │
│ │ 대기 중                 │  │
│ └─────────────────────────┘  │
└───────────────────────────────┘
```

### 5.2 비콘 시작

1. **물건 이름** 입력 (예: 리모컨, 지갑, 안경)
   - 한글/영문/숫자 가능, 최대 16자
2. **광고 시작** 버튼 탭
3. 상태 표시가 다음과 같이 변경:
   ```
   광고 중: IF-리모컨
   (Flipper Zero가 이 신호를 검색합니다)
   ```
4. 알림 영역에 **Item Finder 비콘** 상시 알림이 표시됨

### 5.3 광고 중지

- 앱에서 **광고 중지** 버튼 탭
- 또는 알림에서 앱 아이콘 탭 → 화면에서 중지
- 앱을 종료해도 서비스 알림에서 직접 중지 가능

### 5.4 폰을 물건에 부착

- 폰 뒷면을 3M 양면 테이프 또는 마그네틱 거치대로 물건에 고정
- 화면이 켜져 있을 필요 없음 (꺼져도 광고 지속)
- 충전 케이블 연결 권장 (장시간 사용 시)

## 6. Flipper Zero와의 연동

### 등록 흐름 (본체 측)

1. 본체 키패드에서 `1` (물건 등록)
2. NFC 태그를 Flipper에 태그 → NFC UID 저장
3. 이름 입력 후 `#`
4. **이 앱**이 실행된 폰을 Flipper 근처로 가져옴
5. Flipper가 `IF-`로 시작하는 광고를 우선 탐색
   - 가장 강한 신호의 IF- 비콘 이름을 식별자로 사용
   - 예: `IF-리모컨` 으로 등록됨
6. 본체 LCD에 "등록 완료!" 표시

### 찾기 흐름

1. 본체 키패드에서 `2` → 물건 선택 → `#`
2. Flipper Zero가 `IF-리모컨` 이름의 광고를 360° 스캔
3. 가장 강한 신호 방향으로 서보 모터 고정
4. 비프 주기로 거리 안내

### 다중 물건 운영

여러 공기계에 각각 다른 이름으로 설정 가능:

| 폰 | 광고 이름 | 부착 물건 |
|---|---|---|
| 폰 A | IF-리모컨 | TV 리모컨 |
| 폰 B | IF-지갑   | 지갑 |
| 폰 C | IF-안경   | 안경 케이스 |

Flipper Zero는 광고 이름으로 구분하므로 MAC이 무작위로 바뀌어도 OK.

## 7. 배터리 절약 팁

### 화면 끄기

광고는 화면이 꺼져도 계속됩니다. 사용 직후 화면을 끄세요.

### 배터리 최적화 해제

설정에서 본 앱을 **배터리 최적화 제외** 목록에 추가:

```
설정 → 앱 → Item Finder 비콘 → 배터리 → 제한 없음
```

### 광고 파워 조정 (코드 수정)

`BeaconService.kt`에서 송신 출력 조정 가능:

```kotlin
.setTxPowerLevel(AdvertiseSettings.ADVERTISE_TX_POWER_HIGH)
   // HIGH       → 가장 멀리, 배터리 많이 씀
   // MEDIUM     → 중간 (집안용 권장)
   // LOW        → 가까이만, 배터리 절약
   // ULTRA_LOW  → 매우 약함
```

### 충전기 연결

집안 상시 사용 시 USB 케이블 연결을 권장합니다.

## 8. 트러블슈팅

| 증상 | 원인 | 해결 |
|---|---|---|
| "BLE 광고 미지원" 표시 | 기기가 BLE Peripheral 미지원 | 다른 안드로이드 기기로 교체 |
| 권한 요청이 반복됨 | 권한 거부 후 재요청 | 설정 → 앱 → 권한 모두 허용 |
| 화면 끄면 광고 멈춤 | 배터리 최적화에 막힘 | 위 "배터리 최적화 해제" 참조 |
| Flipper가 비콘 못 찾음 | 광고 이름 미일치 | 본 앱이 정말 광고 중인지 알림 확인 |
| 거리 측정이 부정확 | 폰 방향·금속물체 영향 | 폰을 평평하게, 금속 케이스 분리 |
| 앱 강제종료 후 광고 중단 | 시스템이 서비스 킬 | 자동 시작 앱 목록에 추가 |
| 한글 이름이 깨져 보임 | Flipper 폰트 한글 미지원 | 영문 별명 사용 권장 (예: IF-remote) |

### 디버그 로그 보기

ADB 로그 확인:
```bash
adb logcat -s BeaconService:V MainActivity:V
```
